# Codeword Set — canonical specification (Research Baseline)

## Purpose

This specification defines the canonical **codeword set** `C ⊂ F2^9` for the ASH Pattern System.

The codeword set is the algebraic structure that governs all state transformations, orbit structure, averaging behavior, and branching topology in the ASH model. Every transformation `x' = x ⊕ c` uses a codeword from this set.

> **STATUS: FULLY CLOSED**
>
> The canonical codeword set `C` has been extracted from published research materials. Exact generators, exhaustive enumeration, and code parameters are specified below. There are no remaining open research-closure items for the codeword set.

## Source grounding

The canonical codeword set is established through three layers:

### 1. Source-grounded published vectors

`simulation.py` in the ASH-Model repository (github.com/flynn33/ASH-Model) — the implementation companion to the published ASH research, cited by ashcosmology.net — lists 6 explicit 9-bit vectors described as "adinkra-inspired codewords (linear transformations / SUSY generators)." The associated research paper (LaTeX source in the same repository) confirms the use of "doubly-even self-dual error-correcting codes" as "a linear code `C ⊂ F2^9`."

The 6 published vectors are:

- `(1, 1, 1, 1, 0, 0, 0, 0, 0)` — weight 4
- `(1, 1, 0, 0, 1, 1, 0, 0, 0)` — weight 4
- `(1, 0, 1, 0, 1, 0, 1, 0, 0)` — weight 4
- `(1, 0, 0, 1, 1, 0, 0, 1, 0)` — weight 4
- `(1, 1, 1, 1, 1, 1, 1, 1, 0)` — weight 8
- `(0, 0, 0, 0, 1, 1, 1, 1, 0)` — weight 4

These 6 vectors are the direct source-grounded evidence. The full 16-member set was **not** printed verbatim in the published source.

### 2. Algebraic derivation from published vectors

Rank analysis over `F2` shows the 6 published vectors have rank 4. Two are linearly dependent: `g5 = g1 ⊕ g6`. The 4 independent vectors (g1–g4) form a basis for a 4-dimensional subspace of `F2^9`.

XOR-closure computation from the 4 independent generators produces exactly 16 = 2^4 vectors. This is a deterministic algebraic consequence — given the generators, the complete subgroup is uniquely determined.

### 3. Resulting canonical codeword set

The 16-member set produced by XOR-closure of the source-grounded generators is the canonical codeword set `C`. Its properties ([9, 4, 4] doubly-even, subgroup, 32 orbits) are verified by exhaustive enumeration below.

## Structural properties

### Subgroup

`C` is a subgroup of `(F2^9, ⊕)`: it contains the identity (zero vector), and is closed under XOR. This was previously derived as a minimum consequence of the `T² = T` averaging-operator requirement and is now confirmed by exhaustive enumeration.

### Ninth-coordinate property

Every codeword in `C` has `b8 = 0` (the 9th coordinate is zero for all 16 members). This is a verified structural fact of the complete code, consistent with the earlier observation from published examples.

This property does **not** re-establish the 9th coordinate as a "derived" or "control/parity" dimension in the foundational sense. It is an algebraic property of this specific code embedded in `F2^9`. All 9 coordinates remain structurally equal participants in the state space `F2^9` at the foundational level.

### Relationship to superseded [8,4,4] code

The first 8 coordinates of the 16 codewords in `C` are identical to the 16 codewords of the superseded [8,4,4] extended Hamming code from the earlier 8+1 formalization (see `core-admissibility.pseudo.md`). The canonical 9D code is the same algebraic object embedded in `F2^9` with the 9th coordinate fixed at 0. The superseded formalization treated the 8-bit projection as the foundational object and derived the 9th bit separately; the canonical formalization treats the full 9-bit vectors as foundational.

## Downstream implementation constraints

### Implementations must

- Use exactly the 16-codeword set specified above — no more, no fewer
- Validate that the provided codeword set matches this specification
- Treat `C` as a fixed, canonical constant — not a runtime parameter or implementation choice
- Enforce that all codewords used in transformations are members of `C`

### Implementations must not

- Invent codewords not in the canonical set
- Extend `C` beyond the 16 specified members
- Treat the codeword set as an open parameter that implementations may choose freely
- Use the `b8 = 0` property to re-derive the 9th coordinate as a dependent control/parity bit
